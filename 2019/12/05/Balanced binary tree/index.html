平衡二叉树(Balanced binary tree)
2019-12-05|学习纪录

平衡二叉树(Balanced binary tree)

  • 我们规定左右子树高度差的绝对值不超过1。
  • 定义结点左子树与右子树的高度差为该节点的平衡因子。高度差绝对不超过1。
  • 插入结点若导致了不平衡，则先找到划入路径上离插入结点最近的平衡因子的绝对值大于1的节点A，再对以根A的子树，调整各节点的位置关系，让其重新达到平衡。

LL(左左插入)平衡旋转(右单旋转)

  • 节点A左孩子B左子树上插入新节点。
  • A向右下旋转，作为B的右子树的根节点，B如果有右节点，作为A的左节点。

RR(右右插入)平衡旋转(左单旋转)

  • 节点A右孩子B右子树上插入新节点。
  • A向左下旋转，作为B的左子树的根节点，B如果有左节点，作为A的右节点。

LR(左右插入)平衡旋转(先左后右双旋转)(两次旋转)

  • 节点A左孩子B右子树C上插入新节点。
  • 将C提升到B的位置，B作为左旋转，交换节点位置，C左到B右。
  • C在B位置，对A、C做一次右旋，C提高到A位置，C右到A左。

RL(右左插入)平衡旋转(先右后左双旋转)(两次旋转)

  • 节点A右孩子B左子树C上插入新节点。
  • 将C提升到B的位置，B作为右旋转，交换节点位置，C右到B左。
    • C在B位置，对A、C做一次右旋，C提高到A位置，C右到A左。

平衡二叉树的查找

  • 与给定值进行比较的关键字个数不超过树的深度。该结论可以用于解决平衡二叉树查找所需的最多比较次数，或者树的最大高度。
  • 含有n个节点的平衡二叉树的最大深度为O(log2n)
  • nh = n h-1 + n h-2 + 1

哈夫曼树

  • 树中结点常常被赋一个值，称为该结点的权。

从树根结点到任意结点的路径长度（经过的边数）与该结点上权值的乘积，称为该结点的带权路径长度。

带权路径长度最小的二叉树称为哈夫曼树，也称最优二叉树。

构造哈夫曼树

  1. 新结点作为只有一个结点的二叉树，排成一个森林。
  2. 构造一个新节点，权值最小的点作为新结点的左右子树，并且将新节点的权值置为左右子树的权值之和。
  3. 将新得到的树加入森林。
  4. 重复2，3，知道森林中只剩下一棵树。

哈夫曼编码

  • 每个字符用同样长度的二进制为表示，称为固定长度编码。
  • 不同字符用不等长的二进制位表示，称为可变长度编码。
  • 没有一个编码是另一个编码的前缀，称为前缀编码，例如0，101， 100
  • 将每个字符当做是一个独立的即诶单，权值位它出现的频度（次数），构造哈夫曼树。
  • 所有的字符结点都出现在叶结点中。编码解释为，根至该字符路径上边标记的序列，标记0表示转向左孩子，标记1表示转向右孩子。

各字符对应的编码为：A->11,B->10,C->00,D->011,E->010，有A,B,C,D,E五个字符，出现的频率（即权值）分别为5,4,3,2,1。
